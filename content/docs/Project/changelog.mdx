## 变更类型说明

### 新增 (Added)
<Callout type="success" title="新功能">
  包括新功能、新模块、新API、新增的配置选项和依赖库
</Callout>

- 新功能、新模块、新API
- 新增的配置选项
- 新增的依赖库

### 修改 (Changed)
<Callout type="info" title="功能改进">
  包括现有功能的改进、性能优化、用户体验改进和配置格式变更
</Callout>

- 现有功能的改进
- 性能优化
- 用户体验改进
- 配置格式变更

### 修复 (Fixed)
<Callout type="warn" title="问题修复">
  包括Bug修复、安全漏洞修复和兼容性问题修复
</Callout>

- Bug修复
- 安全漏洞修复
- 兼容性问题修复

### 破坏性变更 (Breaking Changes)
<Callout type="error" title="不兼容变更">
  这些变更可能导致现有代码无法正常工作，需要特别注意
</Callout>

- 不兼容的API变更
- 移除过时的功能
- 配置格式的重大变更

### 移除 (Removed)
<Callout type="warn" title="功能移除">
  已废弃的功能、不再支持的配置选项和移除的依赖库
</Callout>

- 已废弃的功能
- 不再支持的配置选项
- 移除的依赖库
